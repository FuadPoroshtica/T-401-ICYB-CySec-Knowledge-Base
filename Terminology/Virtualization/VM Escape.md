# VM Escape

**Term**: **[VM](Virtual%20Machine%20(VM).md) Escape / [Container](Virtualization%20methods/Container.md) Escape**

**Definition**: An attack where code running inside a [VM](Virtual%20Machine%20(VM).md) or [Container](Virtualization%20methods/Container.md) breaks out of its isolation and gains control of the host or other guests.

**Context/Example**: A malicious program inside a [Container](Virtualization%20methods/Container.md) exploits a kernel bug to get root on the host Linux system—this is the “holy grail” for [Virtualization](Virtualization.md) attackers.

**Related Concepts**: [Hypervisor vulnerabilities](Hypervisor.md), kernel exploit, sandbox escape.
